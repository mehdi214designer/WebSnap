How it works

Capture. Import. Edit.

Three moves. No middleware, no servers, no setup beyond installing the extension and the plugin.

01

Capture from Chrome

Click the WebSnap icon on any page, pick a viewport, hit Capture. A .wsnap file downloads.

02

Open in Figma

Run the WebSnap plugin in Figma, drop the .wsnap file in, and let it build.

03

Edit anything

The page lands as a real Figma frame. Text, fills, gradients, vectors — all live, all editable.

Installation

Up and running in two minutes.

Load the Chrome extension and the Figma plugin. No build step, no dependencies.

Copy git clone https://github.com/mehdi214designer/WebSnap.git
Chrome extension

Load into Chrome

  1. Open chrome://extensions in Chrome.
  2. Toggle Developer mode on (top-right corner).
  3. Click Load unpacked.
  4. Pick the chrome-extension/ folder from the cloned repo.
Optional. To capture local .html files, enable Allow access to file URLs on the WebSnap card.
Figma plugin

Import into Figma

  1. In Figma, open Plugins → Development → Import plugin from manifest.
  2. Pick figma-plugin/manifest.json from the cloned repo.
  3. You're done — run it from Plugins → Development → WebSnap.
The plugin appears under your development plugins. No Community submission required.
Usage guide

A capture, end to end.

From a live URL to an editable Figma frame in well under a minute.

  1. Open the page in Chrome

    Any URL works. Tailwind sites, Framer pages, marketing pages, dashboards — full DOM coverage.

  2. Click the WebSnap icon

    The popup opens. Pick a viewport (1920 / 1440 / 1024 / 768 / 390 / current) and a mode (full page or just the visible area).

  3. Hit Capture

    WebSnap emulates the chosen viewport via DevTools, scrolls through to trigger lazy loads, settles animations, and writes a .wsnap file to Downloads.

  4. Open the Figma plugin

    Run Plugins → Development → WebSnap inside any Figma file. Drag the .wsnap onto the plugin window.

  5. Watch it build

    Progress bar runs as the plugin walks the captured tree. The page lands as a native Figma frame next to your cursor.

  6. Edit anything

    Text is real text, vectors are real vectors, fills are real fills. Move, restyle, swap, take what you need into your own design.
